2. 更换远程仓库协议（推荐）
HTTPS 协议可能因证书或网络限制导致握手失败，可改用 SSH 协议推送（需提前配置 GitHub SSH 密钥）：
先删除现有 HTTPS 远程地址：
bash
git remote rm origin
添加 SSH 远程地址（替换为你的仓库 SSH 地址，在 GitHub 仓库页面获取）：
bash
git remote add origin git@github.com:he2shuang/rasa-test-WSL.git
再次推送：
bash
git push --set-upstream origin master
3. 更新 Git 或依赖库
如果是 WSL 或 Linux 系统，可能因 gnutls 库版本问题导致，尝试更新相关组件：
bash
# Ubuntu/Debian 系统
sudo apt update && sudo apt install --upgrade git gnutls-bin

# CentOS/RHEL 系统
sudo yum update git gnutls

1. 确认并接受主机指纹
当出现 Are you sure you want to continue connecting (yes/no/[fingerprint])? 提示时，输入 yes 并回车，这会将 GitHub 的主机密钥添加到本地信任列表中，避免后续重复验证。
2. 检查 SSH 密钥是否存在
如果还没有生成 SSH 密钥，需要先创建（如果已生成可跳过此步）：
bash
# 生成 SSH 密钥（一路回车使用默认配置即可，无需设置密码）
ssh-keygen -t ed25519 -C "你的邮箱地址"  # 邮箱建议与 GitHub 注册邮箱一致
3. 查看并复制 SSH 公钥
生成密钥后，查看公钥内容并复制：
bash
# 查看公钥（默认路径）
cat ~/.ssh/id_ed25519.pub
输出类似如下内容，全选并复制（包括 ssh-ed25519 开头和邮箱结尾的完整字符串）：
plaintext
ssh-ed25519 AAAAB3NzaC1yc2EAAAADAQABAAABAQC... 你的邮箱@example.com
4. 将公钥添加到 GitHub 账户
登录 GitHub，点击右上角头像 → Settings → 左侧菜单 SSH and GPG keys → New SSH key。
在 Title 中填写一个标识（如 “WSL 设备”），在 Key 中粘贴刚才复制的公钥内容，点击 Add SSH key 保存。
5. 测试 SSH 连接
确认配置正确：
bash
ssh -T git@github.com
如果成功，会显示类似提示：
plaintext
Hi he2shuang! You've successfully authenticated, but GitHub does not provide shell access.
6. 重新推送代码
完成上述步骤后，再次执行推送命令：
bash
git push --set-upstream origin master
